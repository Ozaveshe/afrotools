## Start here: the one thing that matters

**`|| 0` on a data lookup is this codebase's dominant defect.**

```js
var x = country.dataset || {};
var rate = x.rate || 0;        // <- absent data is now "free"
```

It converts *"we don't have this"* into *"the answer is zero / free / Nigeria / visa-required"*, and then renders it **authoritatively** — same styling, same confidence as a researched answer. Nothing throws. Every page returns 200. The tool just says something false.

It was the root of **every P0 found this week**, across three categories and the homepage:

| tool | what it told users |
|---|---|
| roaming-cost | roaming is free (8 of 12 countries) |
| whatsapp-vs-sms | WhatsApp is free (8 of 12) |
| visa-checker | "Visa Required" for 316 of 380 pairs, incl. ECOWAS visa-free routes |
| afrorates | Nigeria's real interest rate is +26.50% (truth: +3.50%) |
| afrostream | 38 of 54 countries are "Nigeria" |
| import-duty | Duty $0, in the default page state |
| japa-calculator | adding a spouse makes relocation $27,736 *cheaper* |

**The cure already exists in your repo.** Copy these; don't reinvent:
- **`tools/currency-converter/`** — `getRate` returns **`null`, never `0`**; null propagates to "Rate unavailable". I could not break it.
- **`tools/vat-calculator/`** — builds its dropdown **from** the data (`if (!d) return`), so the coverage bug is structurally impossible. Its "54 countries" claim is exactly true.
- **`telecom/tv-compare/index.html:256`** — filters the dropdown to countries with data.

**Highest-leverage single action available: a lint rule** banning `|| 0`, `|| 1`, `|| {}` on data lookups in scoring/pricing paths. It would have caught nearly everything below at authoring time.

**The tell:** a dropdown offering more options than the dataset covers. Enumerate the options, check the data, and **execute the code path** — do not eyeball it.

## Second thing: verification in this repo is booby-trapped

I got caught by this **three times**. Learn it free:

- **The build minifies.** Symbol names get mangled; CSS comments are stripped. Grepping production for `_fxRate` or a code comment returns `0` **even when the fix shipped fine**.
- **Grep symbols locally. In production, grep string literals** (`/api/forex`), **code expressions** (`-21)/166`), **or drive the page.**
- `git status` clean ≠ your edit failed. Automations commit with `git add -A`; check `git log -S "<your string>"`.
- **`engines/*.js` is stale vs `engines/src/*.js`** (afroatlas bundle built 2026-03-24 against source updated 2026-07-13). Engine fixes need `engines/src/` **plus** `scripts/minify.js`, or they don't ship.

## Landmines

- **`git commit -a` / `git add -A` will hurt you.** A Codex session runs in this repo and commits the whole working tree — it swept 5 of my telecom fixes into `cf24351 "fix: retire duplicate bank rate scheduler"`, where they remain mislabelled. **Always `git add -- <explicit paths>`.**
- **Untracked module + tracked callers.** `scripts/lib/image-size.js` was untracked while 3 tracked build scripts already `require`d it. A `git commit -a` would have broken every build with `MODULE_NOT_FOUND`. Fixed in `52e25b9`, but **check for this shape again** — a new file in `scripts/lib/` with tracked callers is the pattern.
- **`.claude/` is gitignored.** `government.md`, `transport.md`, `telecom.md` are **local only**. A fresh clone has no rules.
- **Netlify branch deploys appear to be off.** The `netlify.toml` comment promising them is just a comment. `verify-deploy-channel.js` also rejects any branch not in `ops/deploy-channel.json` `releaseBranches` (`main`, `codex/salarypadi-catalog-contract`) — so pushing a feature branch fails CI *by design*, harmlessly.
- **Authority websites go stale.** URA still publishes pre-July-2026 PAYE bands with no effective date; BURS's FAQ still shows old deadlines. **Checking the authority's page can confirm the wrong answer.** Verify against the gazetted Act.
- **OneDrive sync** intermittently throws `ENOENT` on files that demonstrably exist. If a local build fails on a file you can `cat`, that's why — it is not a real defect.
