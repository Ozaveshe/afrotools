# AfroTools Copy Style Guide

Status: working guide for public AfroTools copy. Use this before editing homepage, AI, tools, country pages, API, widgets, sponsor pages, privacy copy, metadata, registries, and generated templates.

## Voice

AfroTools copy should feel:

- practical
- direct
- locally aware
- calm
- confident but not exaggerated
- helpful for African users
- understandable to non-technical users

Avoid:

- hype
- vague AI promises
- unsupported authority claims
- abstract platform language
- investor-style positioning on utility pages
- legal, tax, immigration, medical, financial, or official-source certainty without proof

## Workflow Positioning

AfroTools should read as a practical African workflow platform, not a generic AI chatbot.

Use this behavior ladder when writing homepage, `/ai/`, `Ask AfroTools AI`, tool, API, widget, sponsor, privacy, or empty-state copy:

- ask
- route
- clarify
- prefill
- calculate
- compare
- generate
- export
- verify with sources where needed

Good positioning:

- `AfroTools AI routes your request to practical African tools.`
- `Ask a question, review the suggested workflow, edit the prefill, then calculate or export.`
- `Use source-linked data where available and confirm high-stakes decisions with the official provider.`

Avoid:

- `Chat with an AI that decides for you.`
- `One AI answer for every African problem.`
- `No need to verify.`
- `Fully automated official result.`

## Brand Naming

Use these names consistently:

| Name | Use |
| --- | --- |
| `AfroTools` | The whole platform and established website/tool brand. |
| `AfroTools AI` | The AI front door and command workflow layer. |
| `Ask AfroTools AI` | The user action, input label, and CTA into the command layer. |
| `Africa's Practical AI Hub` | Brand line for the homepage and `/ai/` when the surrounding copy explains routing into practical tools. |
| `AI assist` | Optional model-provider help after explicit consent. |
| `workflow matching` | Deterministic/local routing that does not call a model. |
| `prefill handoff` | Passing safe fields into an existing tool. |

Avoid or phase down:

- `AfroBot` on public marketing surfaces.
- `AI Advisor` as a generic label unless the specific feature calls the existing advisor pathway with consent.
- `AI reasoning` for deterministic routing.
- `AI knows`, `AI will handle`, or `AI decides`.

## Headline Rules

Good headline patterns:

- `Import Duty Calculator`
- `Find scholarships open to African students`
- `Create an ATS-ready CV`
- `African fuel prices, generator costs, and country comparisons`
- `Add useful African calculators to your website`
- `African business data your product can call`

Avoid headline patterns:

- `The best AI platform for everything`
- `The #1 African tools ecosystem`
- `Guaranteed accurate answers`
- `Official calculator for Africa`
- `Complete solution for all countries`

Homepage and `/ai/` may use:

- `AfroTools AI - Africa's Practical AI Hub`

But supporting copy must explain the practical behavior:

- routes prompts into existing tools
- asks clarifying questions
- creates safe prefill handoff
- keeps AI/model assist optional and consent-based
- preserves tool pages as canonical workflows

## Claim Tiers

Use the right label for the evidence available.

| Tier | Label | Allowed when |
| --- | --- | --- |
| A | `official source` | A maintained government, regulator, authority, or statutory source exists and is linked or captured in a source ledger. |
| B | `source-linked` | A credible source URL exists, but the source is not necessarily official. |
| C | `reviewed` | A human or automation checked it on a known date. |
| D | `latest available` | It is the newest snapshot AfroTools has, with a visible freshness date. |
| E | `planning estimate` | It is calculated, modelled, or approximate. |
| F | `user-entered` | The user supplied the value. |
| G | `source needed` | Do not promote as fact yet. |

## Risky Words

Do not use these words unless the proof is present:

- `official`
- `verified`
- `real-time`
- `live data`
- `secure`
- `guaranteed`
- `accurate`
- `best`
- `#1`
- `trusted by`
- `used by millions`
- `complete`
- `all countries`
- `instant approval`
- `AI knows`
- `fully secure`

Preferred replacements:

| Risky | Use instead |
| --- | --- |
| `official data` | `source-linked data`, `authority source where available` |
| `verified` | `reviewed`, `source-linked`, `account-verified`, `contributor-reported` |
| `real-time` | `latest available`, `recent snapshot`, `live endpoint where available` |
| `live data` | `source-backed data`, `cached data with date`, `production endpoint where available` |
| `fully secure` | `runs in your browser where possible`, `sent only after consent` |
| `guaranteed` | `estimated`, `not guaranteed`, `subject to provider review` |
| `accurate` | `based on current source data`, `planning estimate`, `check official source` |
| `best` | `useful`, `popular`, `recommended for`, `compare options` |
| `#1` | remove unless independently sourced |
| `complete` | `practical`, `step-by-step`, `starter`, `covers key checks` |
| `all countries` | `supported countries`, `selected African markets`, exact count if verified |

## AI Copy Rules

Use:

- `AfroTools AI routes your prompt to existing tools.`
- `It asks for missing details before opening the tool.`
- `Optional AI assist runs only after consent.`
- `The tool opens with a safe prefill; you choose when to calculate, save, export, or send anything.`
- `Deterministic routing works without model consent.`

Avoid:

- `AI replaces a lawyer/tax adviser/accountant/immigration consultant/doctor.`
- `AI guarantees the right answer.`
- `AI knows official rules.`
- `AI sends your documents automatically.`
- `AI reasoning` for local keyword matching.

High-stakes AI disclaimer:

Use this pattern for tax, legal, immigration, medical, finance, customs, scholarships, or government workflows:

> Planning guidance only. Confirm decisions with the relevant authority, source, or qualified professional.

## Privacy Copy Rules

Use specific privacy claims:

- `Runs in your browser where possible.`
- `Files stay in your browser for this action.`
- `This workflow uses account sync only if you sign in and save.`
- `AI assist sends your prompt and selected context only after you consent.`
- `Analytics records metadata such as tool id, country code, consent state, or export type.`

Avoid broad claims:

- `fully secure`
- `100% private`
- `no data ever leaves your browser`
- `anonymous` when prompts or inputs may include personal data
- `compliant with every African privacy law`

Safer form for AI:

> AI assist may send your prompt and selected workflow context to AfroTools servers and a configured model provider after you consent. Do not include personal identifiers or sensitive documents unless the workflow explicitly asks and you choose to continue.

## Source And Freshness Copy Rules

Every changing-rate, rule, deadline, fee, or availability workflow should answer:

- What is the source?
- When was it last reviewed or updated?
- Is it official, source-linked, estimated, cached, user-entered, or unknown?
- What should the user verify before acting?

Use:

- `Last reviewed: YYYY-MM-DD`
- `Source-linked estimate`
- `Planning estimate`
- `Stored rates are not labeled official`
- `Confirm filing, payment, or legal decisions with the relevant authority`

Avoid:

- `official rates` without source metadata
- `verified live feed` without current feed proof
- `real-time prices` without a live current source
- `accurate for all countries`

## Metadata Rules

Titles:

- Put the tool or page noun first.
- Add country only when the page is country-specific.
- Keep `AfroTools` at the end.

Descriptions:

- 120 to 160 characters where possible.
- Include the tool name or task.
- Include country/category terms when relevant.
- Add caution through words like `estimate`, `source-linked`, `where available`, or `browser` instead of hype.

Avoid:

- `official`, `verified`, `real-time`, `complete`, `all countries`, or `AI-powered` in metadata unless proof exists.

## Error, Empty, And Loading States

Use calm, useful messages:

- `We could not load this data right now. Try again or use the manual fields.`
- `No matching tool yet. Browse tools or try a more specific country, amount, or document type.`
- `Checking the local tool catalog first.`
- `Opening the tool with prefill. Review before calculating.`
- `AI assist was not contacted because consent was not provided.`

Avoid:

- `Something went wrong`
- `AI failed`
- `No data available` without next action
- `Try again later` without fallback

## Sponsor And Partner Copy

Use:

- `Sponsored placement`
- `Partner link`
- `Request sponsorship`
- `Lead handoff after user opt-in`
- `Placement subject to review`

Avoid:

- implying AfroTools endorses a sponsor's ranking
- hiding sponsored status
- promising lead volume
- promising approval, funding, immigration, or financial outcomes
- putting a sponsor gate in primary exports unless explicitly requested

## API Copy

Use:

- `sandbox examples`
- `dashboard key`
- `reviewed production pilot`
- `production endpoint where available`
- `freshness labels`
- `rate limits`
- `commercial review`

Avoid unless validated:

- `go live`
- `official SDKs`
- `real-time`
- `verified live testing`
- `production data immediately`

## Generated Copy Rules

Before editing generated public text, find the source.

Common sources:

- `assets/js/components/tool-registry.js`
- `scripts/`
- `data/tool-directory.json`
- `data/search-index.json`
- locale JSON and i18n generators
- route templates under category/tool folders

If a source exists, change the source and regenerate. Do not hand-edit generated files first.

## Future Codex Copy Rules

Future Codex prompts that touch AfroTools public copy should follow these rules:

- Preserve SEO and internal discovery keywords. Do not remove important country, tool, category, API, widget, sponsored tool, or country-tool phrases while reducing claim risk.
- Do not change routes, ids, data schemas, calculator formulas, field names, analytics event names, or test selectors unless the user explicitly asks and the change is needed.
- Keep AfroTools as the platform, AfroTools AI as the workflow layer, Ask AfroTools AI as the command input experience, and Africa's Practical AI Hub as the main positioning line where context supports it.
- Describe the AI layer as routing, clarifying, prefilling, calculating, comparing, generating, exporting, and verifying with sources where needed. Do not describe deterministic routing as AI reasoning.
- Keep AI consent off by default. Say private content is not sent to a model unless the workflow asks and the user agrees.
- Use source labels precisely: official source, source-linked, reviewed estimate, user-entered, last reviewed, freshness, and confirm with the official provider.
- Use concise high-stakes disclaimers for tax, immigration, legal, finance, scholarships, import duty, and energy planning. Results are planning aids, not official advice.
- Prefer practical CTAs: `Open tool`, `Calculate now`, `Create CV`, `Find scholarships`, `Compare options`, `Browse countries`, `View API`, `Start with AfroTools AI`, and `Continue without AI`.
- Empty, error, and loading states should explain recovery steps, suggest related tools or categories, and avoid blaming the user.
- Sponsor and API copy must not imply endorsement, guaranteed leads, guaranteed performance, official partnership, or production access unless the implementation and review support it.
- Privacy copy must distinguish browser-local workflows, `sessionStorage` prefill handoff, server workflows, analytics metadata, and optional AI/model processing.
- Update the copy lint/audit helper when new risky claim patterns appear. Do not add heavy dependencies for copy checks.
- Run the content-review audit, targeted checks for touched surfaces, full tests when requested, build when requested, and `git diff --check`.

- required preserved SEO terms
- whether Supabase/live source truth is needed

- whether AI/model consent is part of the workflow
- validation commands to run

Do not add official, verified, real-time, secure, guaranteed, accurate, best, #1, all-countries, instant-approval, or AI replacement claims unless the backing source and validation are present.
