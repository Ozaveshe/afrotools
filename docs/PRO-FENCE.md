# AfroTools Pro Fence

Updated: 2026-05-13

## Rule

Free calculators and public discovery stay reachable to guests. Pro unlocks account-backed operating work, higher-volume convenience, cleaner exports, and reusable handoff workflows. Do not move core public calculator math behind Pro.

## Guest-Reachable Free Surfaces

- Public tool pages, category hubs, country hubs, blog guides, widgets demos, API docs, and business enquiry pages.
- Core calculator inputs and results.
- Educational copy, source notes, and public cross-links.
- Free account sign-up, dashboard entry, and limited saved history where already supported.

## Pro-Unlocked Features

- `/pro/workspace/`, `/pro/apps/`, `/pro/vault/`, `/pro/team/`, `/pro/settings/`, and `/pro/settings/billing/`.
- Account-backed workspaces where a real API or Supabase table exists.
- Expanded saved history where supported.
- Clean PDF exports where the shared PDF gate is wired.
- Reusable handoff notes, packets, and operating checklists.
- Early access to shell apps through `assets/js/components/pro-waitlist.js`.

## AfroPayroll Boundary

The free PAYE and payroll calculators remain public. AfroPayroll Pro extends them into an operating workspace: payroll runs, review packs, saved rows, packet exports, warnings, and audit-style handoff. It does not submit statutory filings, disburse salaries, or prove tax remittance.

## Shell App Boundary

Every non-payroll Pro app must show one of these truthful labels:

- `Local preview only`
- `Coming soon`
- `Account sync pending`
- `Review packet only`

Shell apps must not claim official filing, salary disbursement, bank sync, legal certification, tax approval, cloud sync, team invites sent, or private intelligence verification unless that flow is backed by a real API or live Supabase table with verified access control.

## Indexing

`/pro/` is indexable as the public Pro value and pricing page. Pro work routes are intentionally `noindex, follow`: workspace, apps, vault, team, settings, billing, success, and cancel.

Regenerate the gate coverage ledger after Pro route changes:

```powershell
node scripts/audit-pro-gate-coverage.js
```
