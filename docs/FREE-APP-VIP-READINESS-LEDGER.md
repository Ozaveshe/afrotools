## Serious public-app deployment checklist

Each app receives this checklist independently; a shared component pass does not automatically promote every page.

- Product truth: one clear user job, correct scope, no duplicate/dead workflow, no unsupported AI/API/live/official/compliance claim, and explicit limitations.
- Calculation/function: pure or isolated logic where practical, known fixtures and boundaries, invalid/stale/error clearing, fail-closed classification, API/widget/locale parity, and protected formula review for material money logic.
- Source confidence: primary authority first for legal/tax/government facts; row-level source and checked/effective dates for changing data; conflict disclosure; freshness threshold; planning disclaimer; no silent fallback presented as current.
- Mobile/layout: 320, 360/375 and 768px; no horizontal overflow, clipping, tiny controls, sticky collisions or compressed tables; 200% text reflow and safe-area behavior where relevant.
- Visual design: calm task hierarchy, consistent tokens, meaningful empty/loading/error/result states, readable tables/charts, high-quality light and dark surfaces, reduced-motion support and no decorative clutter that competes with the task.
- Accessibility: visible labels, logical heading order, keyboard completion, visible focus, 44px practical targets, semantic status/error regions, modal focus/Escape behavior, contrast, non-colour cues and useful image alternatives.
- Privacy/security: minimum data collection, local-first sensitive inputs, no raw PII/financial/legal/health content in analytics/logs/URLs/screenshots/storage, explicit consent before network/AI transfer, safe external links and no secret/internal artifact exposure.
- SEO: unique bounded title/description, served self-canonical, index/noindex intent, reciprocal hreflang, correct OG/social image, valid breadcrumbs and internal links, and sitemap inclusion only for intended canonical routes.
- AI-search: truthful registry name/description, concise answer-first copy, useful FAQ only when visible, valid WebApplication/Dataset/Article schema as appropriate, `isBasedOn`/date/source ownership for factual tools, source-coupled AI context, and no hand-typed dynamic facts outside the owner.
- Export/PDF: feature exists only when useful, local and ungated by default, correct filename/content/encoding, parser-readable production-shaped fixture, no sensitive leakage, and copy/CSV/TXT/JSON/print labels match actual behavior.
- Reliability/performance: no console/page errors, deterministic loading/failure states, offline boundary stated, asset and third-party failures handled, reasonable image/font/script weight, no layout shift regression and no unnecessary network request from tool inputs.
- Analytics/monetization: non-PII metadata only, stable event names, sponsor/affiliate/lead labels, explicit opt-in for handoff, and no paywall/lead/account gate in the primary local export path unless the product contract requires it.
- Release proof: focused tests, screenshot inspection, syntax/diff/link/SEO/i18n gates, source/registry/generated-owner replay, build and `dist` audit, security scan, exact-SHA preview, then exact-SHA production deploy and representative live-route verification. Local, artifact, preview and live proof remain separate ledger states.
