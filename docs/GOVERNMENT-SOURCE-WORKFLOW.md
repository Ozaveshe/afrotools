# Government Source Workflow

The `/government/` hub is an independent AfroTools surface, not an official government service. Treat official portals and regulator pages as the source of truth before changing fees, forms, eligibility, or timelines.

## Source Files

- `government/index.html` is the public hub.
- `assets/css/government.css` is the hub styling.
- `data/government/official-sources.json` maps tools to official source pages.
- `data/government/source-status.json` is generated by the source checker.
- `reports/government-source-ledger.md` is generated evidence for the latest run.
- `scripts/update-government-source-ledger.js` probes official sources and refreshes the hub source-summary block.

## Commands

```bash
npm run government:sources
npm run government:sources:check
```

Use `npm run government:sources` when refreshing the hub. It writes the status JSON, the Markdown report, and the source-summary block in `government/index.html`.

Use `npm run government:sources:check` for CI or release review. It validates the manifest, probes sources, and fails only on broken required sources without writing files.

For the first run after changing source-classification logic, run `node scripts/update-government-source-ledger.js --baseline` so the checker records a clean comparison point without treating the classification change itself as a public-source change.

## Review Rules

- Use only official government, regulator, treasury, electoral, civil-registration, immigration, pension, welfare, or public-services portals for primary facts.
- Treat login-gated, blocked, or rate-limited sources as manual-review required, not as proof that a fact is false.
- If a source content hash changes, inspect the official page or document before changing a tool.
- Do not replace an official source with a blog, agent, travel agency, SEO page, or scraped snippet unless it is clearly marked as secondary context.
- Keep public copy honest: tool counts must match visible linked routes and schema `numberOfItems`.
- Run `npm run check-links` after changing hub links or routes.

## Automation Model

The recurring job should:

1. Run `npm run government:sources`.
2. Read `data/government/source-status.json` and `reports/government-source-ledger.md`.
3. Separate broken official sources from changed official sources.
4. Patch tool data only after a human-readable source change is confirmed.
5. Run `npm run government:sources:check`.
6. Run `npm run check-links` when hub routes, source links, or schema changed.

Keep source-monitor output separate from unrelated dirty-tree changes.
