# Transport Source Workflow

The `/transport/` hub is a planning surface for transport, fleet, vehicle import, logistics, and city-travel workflows. It is not an official government or operator service. Treat official authority and direct operator pages as the source of truth before changing prices, tariffs, documents, routes, duties, or compliance requirements.

## Source Files

- `transport/index.html` is the public hub.
- `assets/css/transport.css` is the hub styling.
- `data/transport/official-sources.json` maps every hub tool to official source pages.
- `data/transport/source-status.json` is generated by the source checker.
- `reports/transport-source-ledger.md` is generated evidence for the latest run.
- `scripts/update-transport-source-ledger.js` probes official sources and refreshes the hub source-summary block.

## Commands

```bash
npm run transport:sources
npm run transport:sources:check
```

Use `npm run transport:sources` when refreshing the hub. It writes the status JSON, the Markdown report, and the source-summary block in `transport/index.html`.

Use `npm run transport:sources:check` for CI or release review. It validates the manifest, probes sources, and fails only on broken required sources without writing files.

For the first run after changing source-classification logic, run `node scripts/update-transport-source-ledger.js --baseline` so the checker records a clean comparison point without treating the classification change itself as a public-source change.

## Review Rules

- Use official transport authorities, customs agencies, fuel regulators, port authorities, road agencies, aviation regulators, city authorities, and direct operator pages for primary facts.
- Treat blocked, rate-limited, login-gated, or JavaScript-heavy portals as manual-review required.
- If a source content hash changes, inspect the official page or document before changing a tool.
- Use operator pages for operator availability, fare models, shipping rules, and route availability. Use government or regulator pages for legal requirements, duties, tolls, public fees, and compliance.
- Keep public copy honest: tool counts must match visible linked routes, JSON-LD `numberOfItems`, and the source manifest.
- Run `npm run check-links` after changing hub links or routes.

## Automation Model

The recurring job should:

1. Run `npm run transport:sources`.
2. Read `data/transport/source-status.json` and `reports/transport-source-ledger.md`.
3. Separate changed official sources from blocked/manual sources and broken required sources.
4. Patch tool data only after a human-readable source change is confirmed.
5. Run `npm run transport:sources:check`.
6. Run `npm run check-links` when hub routes, source links, or schema changed.

Keep source-monitor output separate from unrelated dirty-tree changes.
