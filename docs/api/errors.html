Error Handling

The AfroTools API uses conventional HTTP status codes and returns structured JSON error objects.

Error Response Format

All errors return a JSON object with success: false and an error object containing a machine-readable code, a human-readable message, and a link to the relevant docs page.

HTTP Status Codes

CodeMeaningDescription
200OKRequest succeeded. Response body contains the requested data.
400Bad RequestThe request was malformed or missing required parameters.

403ForbiddenAPI key is valid but lacks permission for this resource or plan feature.

429Too Many RequestsRate limit exceeded. Slow down and retry after the period indicated.
500Internal ErrorSomething went wrong on our servers. These are automatically reported.
503Service UnavailableAPI is temporarily down for maintenance. Check status page.

Troubleshooting

401 Unauthorized

Make sure you are including your API key in either the x-api-key header or as the api_key query parameter. Keys start with afro_live_ or afro_test_.

429 Rate Limit Exceeded

You have exceeded your plan's daily call limit. Check the X-RateLimit-Remaining and X-RateLimit-Reset response headers. Upgrade your plan or implement exponential backoff.

400 Bad Request

Check the error code field for specific details. Ensure all required parameters are present and correctly formatted. Country codes must be 2-letter ISO codes.

500 Internal Error

These are rare and are automatically reported to our engineering team. If the issue persists, contact api@afrotools.com with the request details and timestamp.
