# Registry and Count Source Inventory

This inventory records every repository-level owner or projection found in the canonical-registry audit. “Owner” means the file is allowed to define a domain fact. “Projection” means it must be regenerated or validated from an owner. “Consumer” means it may display/filter the data but must not invent totals.

## Canonical ownership

| Domain | Canonical owner | Normalized output | Notes |
|---|---|---|---|
| Tool records, tool status, tool-family weights, category IDs | `assets/js/components/tool-registry.js` | `data/registry/canonical-registry.json` | Legacy browser shape is normalized into the explicit canonical tool schema. Four duplicate-route rows are declared aliases in policy. |
| Category metadata and category hubs | `AFRO_CATEGORIES` in `assets/js/components/tool-registry.js` | Canonical `categories` records and `tools.category.*.published` selectors | Page-local category copy may add presentation metadata but cannot define category membership or totals. |
| Country identity, country routes, currencies | `data/registry/countries.json` | Canonical `countries` records and generated `assets/js/data/african-countries.js` | Exactly 54 jurisdictions. `ALL` is a regional applicability sentinel, never a 55th country. |
| Site locale states | `data/registry/locale-manifest.json` | Canonical `locales` records and generated browser manifest | English is default; French and Swahili are launched; Yoruba and Hausa are partial; Igbo and component-only Portuguese/Arabic are planned with no public locale prefix. |
| Widget records and iframe destinations | `widgets/WIDGET-REGISTRY.js` | Canonical `widgets` records | The widget build owns record generation; canonical validation owns IDs, destinations, categories, and semantic counts. |
| API quota plans | `netlify/functions/_shared/api-plans.js` | Namespaced canonical `api:*` plans | The legacy `starter` input is an alias of the `growth` tier and is not counted as a fifth API plan. |

## Database query audit

No live Supabase query supplies the tool, widget, category, country, locale, plan, publication, or headline totals in this scope. Supabase queries found in the repository concern user/account data, saved work, content datasets, products/prices, automation, and other runtime features. They are not catalog membership owners. The canonical registry build is repository-local and performs no live project action.

## Semantic definitions

- **Tool record:** one stable normalized registry row. A localized route is its own record. A redirect alias remains auditable but is not published/indexable membership.
- **Canonical published tool:** a published, non-deprecated, non-alias record with a working repository destination.
- **Live tool experience:** a unique English published destination plus intentionally declared hidden country variants from weighted tool families.
- **Widget:** a stable published widget record with a working iframe destination.
- **Country:** one canonical African jurisdiction in `countries.json`; `ALL` is a regional sentinel.
- **Site language:** a locale in catalog policy marked public. AI-supported languages are a separately labeled capability.
- **Plan:** a namespaced API quota tier or product currency/interval option; the two namespaces are never added together under an ambiguous “plans” label.
