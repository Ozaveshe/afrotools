Engineering Fix

The broken FX v1 path was repaired at the right layer.

Live smoke showed the documented FX route returning 500 while the underlying forex data lane was healthy. The handler now reads the shared cached data-store path instead of making a separate service-role Supabase query.

  • Removed the brittle direct Supabase service-key dependency from the public FX v1 route.
  • Reused the same forex-latest live-data source already powering the working AfroTools forex lane.
  • Kept sandbox behavior intact for afro_test_* keys and kept anonymous free testing for live FX examples.
