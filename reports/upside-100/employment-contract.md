# Employment Contract Builder — Audit

Tool: https://afrotools.com/tools/employment-contract/
Source: `C:/Users/Oza/Documents/afrotools/tools/employment-contract/index.html`
Category: Legal · Reviewed 2026-07-13

## What it does
Client-side employment-contract generator. User fills employer/employee/compensation fields and toggles benefit/protective clauses (pension, HMO, NDA, IP, non-compete, maternity, 13th month, remote, etc.). `generate()` assembles a clause list from an inline `LAWS` object and renders an on-page contract preview. Output actions: **Print** and **Download as Text (.txt)** only. No PDF, no DOCX, no e-signature.

## Jurisdiction coverage (important mismatch)
- **Interactive builder supports only 8 countries** — `LAWS` object (line 371) and the `#country` select (line 136): NG, KE, GH, ZA, TZ, UG, RW, ET.
- Hero copy claims **"Now covering all 54 African countries"** (line 107) and there are 54 dedicated country landing links (lines 111-128, 365).
- The 54 country pages are **generated** by `scripts/gen-employment.js` from `C:/Users/Oza/Downloads/afrotools-270-master-data.js` (external data file). They carry better titles + WebApplication/BreadcrumbList JSON-LD, but the hub builder itself only truly localises 8. Overclaim = trust/accuracy risk.

## H1 finding (the flagged issue) — EXACT
- **On initial static load there is exactly ONE `<h1>`**: `"Employment Contract Builder"` in the hero (line 107). It is **not missing and not duplicated at load.**
- The SEO section's lead heading `"Employment Contract Builder for African Businesses"` is an **`<h2>`** in source (line 177) — a WebFetch render mislabels it as h1; it is NOT a second static h1.
- **The real duplicate:** `generate()` injects `'<h1>Employment Contract</h1>'` into the preview (line 416) when the user clicks Generate. There is dedicated CSS `.agreement-preview h1` (line 39). After interaction the live DOM contains **two `<h1>`s**. Crawlers that don't click won't see it, but it is a genuine semantic/accessibility duplicate-H1 defect and matches the audit flag.

## SEO audit
- **Title** (line 8): `"Employment Contract Builder — African Labour Law Templates"` (~57 chars). Acceptable length but weak: no brand suffix, no high-intent modifiers ("Free", "Word/PDF"). Country pages append `| AfroTools` — hub does not.
- **Meta description** (line 9): good, ~200 chars, keyword-rich.
- **JSON-LD:** only **BreadcrumbList** present (lines 69-94). **Missing FAQPage** despite 5 visible FAQs (lines 187-191) — high-value miss. **Missing WebApplication/SoftwareApplication** on the hub (country pages have it; hub does not). No HowTo.
- **Canonical** present (line 99); hreflang en/fr/sw/x-default present (100-103). Good.
- Headings otherwise well structured; heavy generated legal-copilot/competitor/action-pack sections add depth (and some near-duplicate boilerplate).
- **Brand token drift:** page uses `--blue:#1d4ed8` (line 17), not the standardised brand `#0062CC`.

## UI/UX audit
- Clean form→preview flow, min-wage live check (`checkMinWage`), sensible defaults.
- **No PDF/DOCX export** — only print + plaintext; competitors offer Word + PDF.
- **No clause explanations/tooltips** — user toggles NDA/non-compete/IP with no "why/when".
- **No worker-classification step** — the page's own Legal Copilot preaches "Confirm worker classification before drafting" and "start with classification," but the builder never asks employee vs contractor/intern/fixed-term.
- Signature blocks exist (lines 425-428) but no e-sign / fillable readiness.
- Mobile: form-grid collapses to 1 col at 600px (line 27); preview padding 2.5rem may crowd 375px. Print CSS is solid.
- **Disclaimer gap:** "not legal advice / consult a lawyer" appears in FAQ (line 187) and action-pack (line 346) but **NOT on/under the generated contract itself** — output footer only says "Generated by AfroTools" (line 429).

## Competitor gaps (LegalZoom, Rocket Lawyer, LawDepot; Deel/Oyster for Africa HR)
1. **Word (DOCX) + PDF export** — all three offer downloadable Word/PDF; this tool has none.
2. **Jurisdiction-aware clause logic at depth** — LawDepot adjusts clauses per state; here only law name/currency/pension/leave label swap across 8 countries, clause bodies are generic.
3. **E-signature / document management** — Rocket Lawyer sign + "Document Defense"; no e-sign here.
4. **Attorney-review upsell / guided interview** — none.
5. **Worker-classification wizard** (Deel/Oyster core) — absent.

1. **Duplicate H1** → `tools/employment-contract/index.html` line 416: change `'<h1>Employment Contract</h1>'` to `'<h2 class="doc-title">Employment Contract</h2>'`; update CSS line 39 selector `.agreement-preview h1` → `.agreement-preview .doc-title`. Keeps one document-outline H1.
2. **Add FAQPage JSON-LD** → same file, add a `<script type="application/ld+json">` FAQPage block mirroring the 5 existing FAQs (lines 187-191).

4. **Output disclaimer** → after line 429 in the generated `innerHTML`, append a visible "Template only — not legal advice; have a qualified lawyer review before signing" line.
5. **Fix the "54 countries" overclaim** → line 107 hero copy: soften to "8-country smart builder + dedicated guides for all 54," OR expand the `LAWS` object. Also brand color `#1d4ed8` → `#0062CC`.
6. **Title** → line 8: e.g. `"Free Employment Contract Builder (Word & PDF) — African Labour Law | AfroTools"`.

### B. Feature upgrades vs competitors
- DOCX + PDF export (replace/augment `.txt`).
- Expand interactive jurisdiction coverage beyond 8; source min-wage from live data / the Minimum Wage tool instead of hardcoded values.
- Clause explanation tooltips.

- E-signature / fillable-signature readiness.

### C. Watch-outs
- **54 country subpages are GENERATED** by `scripts/gen-employment.js` (reads an external file in `Downloads/`). Do not hand-edit `tools/employment-contract/<country>.html`; regenerate.
- The `LEGAL-WORKFLOW-COPILOT`, `LEGAL-COMPETITOR-CHECK`, `LEGAL-DEEP-IMPROVEMENT`, `LEGAL-ACTION-PACK` blocks (lines 193-352) are injected by legal-section scripts (`enhance-legal-section-pass.js`, `verify-legal-workflow.js`); `legal-workflow-copilot.js` is shared across legal tools — edits may be overwritten and affect siblings.
- **Hardcoded min-wage data will go stale** and is already inconsistent: `LAWS` Ghana `minWage:400` vs FAQ "GH¢430-450" (line 190); ZA `minWage:4413` monthly approximation vs statutory R27.58/hour; FAQ framed "As of 2024" (now 2026). Legal-accuracy red flag.
