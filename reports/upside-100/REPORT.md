## WAVE 1 — tools 1–10 (audited 2026-07-13)

### ✅ Systemic fix (affects 97 pages, not just Wave 1)
**Bug:** `scripts/add-webapplication-schema.js` `inferCategory()` tested `relPath.includes('html')`, which matches `index.html` on *every* page — so any tool not caught by an earlier finance/health/education keyword fell through to `applicationCategory: "DeveloperApplication"`. 119 non-dev tool pages (bride-price, car-loan, age-calculator, savings-goal, scientific-calc…) were mis-tagged as developer tools in their JSON-LD — a real structured-data defect.
**Fix shipped:**
- New `scripts/fix-webapplication-category.js` (conservative: only flips pages currently `DeveloperApplication` whose slug-based re-classification differs; genuine dev tools kept). **97 pages corrected** → 71 WebApplication, 15 FinanceApplication, 7 DesignApplication, 4 EducationalApplication. All JSON-LD re-validated as parseable.
- Hardened the root generator `add-webapplication-schema.js` (slug-based matching, dropped the `html`/`url` catch-all keywords) so it cannot recur.
- **Follow-up:** the FR mirror (`fr/tools`, `fr/crypto`) has the same defect — deferred to run through the FR pipeline.
