Kanuni za kupanga

Zana hutumia kanuni hizi:

  • Clauses kuu kama SELECT, FROM na WHERE huanza kwenye mistari mipya
  • JOIN hupata mstari wake
  • Orodha ya columns hupangwa chini ya SELECT
  • Masharti ya AND/OR hupangwa chini ya WHERE
  • Keywords hubadilishwa kulingana na chaguo lako
  • Nafasi karibu na operators husawazishwa

Njia za mkato

  • Ctrl+Enter — Panga SQL
  • Tab — Ongeza indentation kwenye textarea

Mazoea mazuri ya SQL

  • Tumia aliases kwa majina ya tables
  • Weka SQL keywords kwa mtindo mmoja
  • Weka column moja kwa mstari kwenye SELECT ndefu
  • Panga subqueries kwa indentation
  • Weka comments kwenye logic ngumu
  • Tumia alias names zinazoeleweka

SQL formatter ya haraka kwa Kiswahili

Tumia zana hii kusafisha query ndefu kabla ya code review, debugging au kushiriki mfano na timu. Bandika SQL, chagua mtindo wa keywords na indentation, kisha pata output inayosomeka vizuri bila kuituma kwa server.

Mpangilio mmoja husaidia timu kuona JOIN, WHERE, GROUP BY na ORDER BY kwa haraka. Hii haibadilishi logic ya query; inabadilisha nafasi, mistari na mtindo wa keywords ili iwe rahisi kukagua.

Inafaa kwa SELECT, INSERT, UPDATE, DELETE na DDL kama CREATE TABLE au ALTER TABLE. Kwa queries zinazogusa malipo, payroll, kodi au data nyeti, tumia output kama msaada wa review na uhakiki performance, indexes na permissions kwenye database yako.

Maswali ya mara kwa mara

Je, formatter hubadilisha logic ya SQL?

Hapana. Inabadilisha whitespace, line breaks na mtindo wa keywords tu. Majina ya tables, columns na values hubaki kama ulivyoyaandika.

Ni SQL dialects zipi zinafaa?

Inasaidia syntax ya kawaida inayotumika kwenye MySQL, PostgreSQL, SQLite, SQL Server na Oracle. Keywords maalum za dialect fulani zinaweza kuhitaji review ya ziada.

Je, SQL yangu hutumwa AfroTools?

Hapana. Formatting hufanyika kwenye browser kwa JavaScript. Usibandike secrets, tokens au data nyeti kama mazingira yako hayako salama.

Mbinu

Tunatumia kanuni za formatting, indentation, capitalization na line breaks kusafisha SQL bila kubadili majina ya tables, columns au values.

Mipaka

Matokeo ni makadirio tu ya usomaji. Hayathibitishi query iko salama, performant au sahihi kwa database yako. Hakiki na DBA au reviewer kabla ya production.

Chanzo na faragha

Chanzo cha output ni SQL unayoingiza kwenye browser. Tarehe ya ukaguzi: 2026. Hakiki chanzo, schema na execution plan kabla ya kuendesha query halisi.
