    Méthodologie

    Le simulateur lit le code court, l'écran d'accueil, les options et la sequence de test. Il genere un transcript CON/END, signale les formats suspects, les options dupliquees, les prompts longs et les sequences incompletes.

    Limites et avertissement: simulation non officielle. Vérifier avec votre agregateur, opérateur, banque ou regulateur avant lancement public.

    Questions fréquentes

    Quelle différence entre CON et END ?

    Dans beaucoup d'intégrations, CON indique que la session attend une autre saisie, tandis que END indique que le parcours peut se terminer. Votre passerelle peut utiliser ses propres conventions.

    Le simulateur stocke-t-il mes données ?

    Non par défaut. Le bouton de sauvegarde enregistre seulement le dernier transcript dans localStorage de ce navigateur.

    Ce test remplace-t-il la certification telco ?

    Non. Il aide à préparer le parcours, mais les tests finaux doivent être faits avec l'opérateur, l'agregateur et les règles du pays cible.
