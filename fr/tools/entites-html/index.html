Méthodologie

Methodology: le mode encoder remplace les caractères reserves HTML et, selon le style choisi, convertit aussi les caractères non ASCII en entités numériques ou hexadecimales. Le mode decoder laisse le moteur HTML du navigateur reconvertir les entités en texte.

  • Regles appliquees: &, <, >, guillemets et apostrophes sont traites en mode minimal.
  • Avertissement: l'encodage de texte ne remplace pas une stratégie XSS complete. Vérifiez le contexte HTML, attribut, URL, CSS ou JavaScript.
  • Confidentialité: les saisies restent locales; aucune requete serveur n'est nécessaire pour convertir.
  • Fraîcheur: références et schema de page revus en 2026.

Questions fréquentes

Entités nommees ou numériques ?

Les entités nommees sont plus lisibles quand elles existent. Les entités numériques couvrent davantage de caractères Unicode.

Pourquoi & doit-il être encode en premier ?

Parce qu'une entité commence par &. Si vous encodez & après les autres caractères, vous pouvez double-encoder une sortie déjà transformee.

Le résultat est-il sur pour tout contexte ?

Non. Un texte encode pour du contenu HTML n'est pas automatiquement adapte a un attribut, une URL, du CSS ou du JavaScript.
