Méthode et limites

Dernière vérification : 2026. Références utilisées pour le comportement navigateur moderne.

  • Encoder URI convient à une URL déjà structurée : les séparateurs d'URL restent lisibles.
  • Encoder composant convient à une valeur isolée : les espaces, accents, &, =, ?, / et # sont protégés avant insertion dans une URL.
  • Décoder accepte les espaces écrits en + dans les chaînes de requête, puis signale les séquences % incomplètes ou invalides.
  • Avertissement. Cet outil aide au diagnostic et au formatage. Vérifiez les URL signées, les liens de paiement, les redirections sensibles et les secrets API dans leur système d'origine.

Pourquoi mon décodage échoue-t-il ?

Un signe pourcentage doit être suivi de deux caractères hexadécimaux, par exemple %20. Une valeur comme %2 ou %ZZ est invalide et déclenche une alerte au lieu de casser l'outil.

Quand utiliser le mode composant ?

Utilisez-le pour une valeur unique : terme de recherche, identifiant, chemin utilisateur, URL de retour ou paramètre API. C'est le choix le plus sûr quand la valeur contient &, =, ?, / ou #.
