    Méthodologie et limites

    La méthode utilise l'API Web Crypto du navigateur quand elle est disponible. Pour le texte, l'outil encode la chaine en UTF-8 puis calcule le digest. Pour les fichiers, le fichier est lu en ArrayBuffer localement, puis le digest est converti en hexadecimal.

    • Dernière vérification des sources: 2026-06-17.
    • SHA-256 est le choix recommande pour une vérification d'integrite courante.
    • SHA-1 est affiche uniquement pour compatibilité avec d'anciens checksums.
    • Le fichier ou le texte ne quitte pas le navigateur pendant le calcul de base.

    Questions fréquentes

    Pourquoi MD5 n'est-il pas propose ici ?

    Cette version utilise Web Crypto natif, qui expose les familles SHA mais pas MD5. Cela evite de charger une bibliotheque externe pour un algorithme deconseille en sécurité.

    Comment vérifier un fichier telecharge ?

    Sélectionnez le fichier, choisissez l'algorithme indique par le fournisseur, collez le hash attendu et comparez le statut.

    Le hash prouve-t-il qu'un fichier est sur ?

    Non. Un hash identique confirme surtout que le fichier correspond a une empreinte attendue. Il ne remplace pas une signature, une source fiable ou une analyse de sécurité.
