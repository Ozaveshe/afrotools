Why these plans exist.

AfroTools API is a B2B data product for teams that need African tax, VAT, FX, fuel, central-bank, and country data inside their own software. The free tier reduces friction. Paid tiers add support, data review, data updates, and integration work.

Pick the plan by product stage.

Use the free tier for technical validation. Choose Growth for one product workflow, Pro when volume or launch support matters, and Enterprise/custom when procurement or custom data scope is required.

Pricing notes.

Can teams use production data immediately?

Free verified keys can call live endpoints where the route supports production data. Sandbox keys return deterministic examples and should not be used for production decisions.

Why are paid tiers pilots?

Higher-volume API plans start with a reviewed activation step so data scope, endpoint fit, volume, support expectations, and commercial terms are confirmed before production use.

Are the API prices final?

Growth and Pro are public pilot prices for sales conversations. Enterprise/custom is negotiated based on volume, data scope, support, and procurement requirements.

What should a team test first?

Start with PAYE, VAT, FX, fuel, central-bank rates, and country metadata. Check the docs and API status before proposing production use.
