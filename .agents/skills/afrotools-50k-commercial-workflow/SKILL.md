## Operating Rule

AfroTools is sales-led, SEO-assisted. Every new feature or page improvement should support revenue, backlinks, email capture, partner distribution, or a verified user/release need.

## Priority Surfaces

- Commercial pages: `/widgets/`, `/sponsored-tools/`, `/custom-calculators/`, `/media-kit/`, `/api/`, `/business-enquiry/`
- Widget technical routes: `/widgets/demo/`, `/widgets/embed.js`, `/embed/`, `/widgets/iframe/`
- Business subscriptions: `/pro/`, `/pricing/`, `/pro/workspace/`, `/pro/apps/`
- Priority money tools listed in `docs/AFROTOOLS-50K-EXECUTION-HANDOFF.md`
- Internal execution OS: `mc-7a2f9x.html` 50K panel for post-deploy QA, local-only first-300 prospect tracking, copyable proposal/outreach templates, and the 7-day execution checklist.
- Segment pages: `/for-accountants/`, `/for-hr-payroll/`, `/for-fintechs/`, `/for-schools/`

## Workflow

1. Inspect the target source route and the matching workflow doc.
2. Preserve existing shared systems: `capture-b2b-lead.js`, `b2b-enquiry-form.js`, `<afro-business-cta>`, widget embed routes, API tier honesty, and Pro data-boundary labels.
3. Improve buyer utility, not generic marketing copy.
4. Keep tax, payroll, API, customs, remittance, and compliance copy honest about estimates, pilots, and backend readiness.
5. Preserve source context through commercial CTAs and B2B forms: offer, tool, source route, prospect segment, CTA type, safe referrer, and UTMs.
6. Keep internal prospect and checklist persistence local-only unless a safe backend is explicitly requested and implemented through validated server routes.
7. Run the narrowest validation that proves the changed surface.
8. Report baseline repo debt separately from net-new issues.

## Validation

Use the command set in `docs/AFROTOOLS-50K-EXECUTION-HANDOFF.md` for the surface touched. For small HTML/content batches, a targeted link/DOM check is acceptable when full repo checks are noisy.
