# V4 Generated Output Review

## Does Generated Output Belong In Git?

Mixed answer:

- `dist/`: no. It is ignored by `.gitignore` and should never be committed.
- Root static HTML, minified assets, generated JSON, sitemaps, and post-processed pages: yes, this repo often tracks them, but they must be separated from source edits.
- Audit artifacts: optional. Keep as evidence only unless the team intentionally tracks release reports.

## Build Artifact Fix

`dist/audit-results/` was present after failed build attempts. That is a publish-surface bug.

Fix:

- Added `audit-results` to `BLOCKED_TOP_LEVEL_DIRS`.
- Added `audit-results` to deploy artifact forbidden verification.
- Changed `build-dist.js` to clear contents of `dist/` instead of deleting the root folder.

Verification:

- `npm run build:deploy`: pass.
- `npm run audit:dist`: pass.
- `npm run security:scan`: pass.

## Cleanup Recommendation

Do not delete generated files blindly. For human review:

```bash
git status --short
git diff --stat
git diff -- scripts/build-dist.js
git diff -- reports/mobile-audit.md reports/mobile-network-smoke.md
```

Then package:

1. `scripts/build-dist.js` as the source packaging fix.
2. v4 audit logs/reports as evidence.
3. Generated report outputs only if the repo intentionally tracks updated audit snapshots.
