Evidence-gated API

The public endpoint applies the same strict policy-rate filter as this page. Coverage metadata distinguishes candidate, verified and withheld rows. A country without complete evidence is not returned as current.

GET /api/rates?metric=policy_rate
GET /api/rates?country=KE
GET /api/rates?metric=inflation
Deliberately absent: MPC calendars and T-bill/bond tables. The committed snapshot currently has no qualifying rows for those fields, so AfroRates does not render empty features as if they were populated.

Evidence contract

A displayed row needs a numeric 0–100% rate, a recognised official central-bank HTTPS host, a decision date, verification no more than 45 days old and inclusion in the snapshot’s verified-code list. If both API and snapshot fail, all rates stay hidden.

Freshness is per row

The dataset generation time does not make every record current. AfroRates shows the decision date and verification date for each row so you can judge its age and open the issuing bank before acting.

Privacy and exports

Search, sorting, CSV and PDF generation stay in your browser. AfroRates does not send your search to AI, analytics or a user account. Exports contain public reviewed data only.

Questions people ask

Are AfroRates values live?

No. This is a reviewed committed snapshot. A central-bank announcement can supersede it at any time.

Why are some African countries missing?

Their committed rows did not pass the evidence contract. Missing is safer than presenting an unsupported value as current.

Is the gap an official real interest rate?

No. It is a rough cross-period comparison, not an official measure, investment return or borrowing recommendation.
