Security Model
QPDF WebAssembly applies PDF standard security locally in the browser. Files and passwords are not uploaded to AfroTools.
  • AES-256 protect output.
  • User and owner password support.

Important
PDF permission flags restrict actions in compliant readers, but they are not the same as document rights management. Keep a clean original and store passwords securely.
Protection limits

Protect a copy, then test the password in another reader

Password protection and unlock attempts run locally. This tool does not recover unknown passwords, store passwords, or guarantee that every PDF reader enforces permissions the same way.

Local-first PDF protection

Reviewed 2026. Disclaimer: verify with the recipient and another PDF reader before relying on protection. The shared PDF account gate does not upload your files.

Methodology

  • Uses local PDF encryption/decryption support to create protected copies or unlock PDFs when the correct password is supplied.
  • Password strength feedback is a local estimate and does not check breach databases.
  • Permission flags can be interpreted differently by PDF readers; encryption strength and compatibility should be tested.

Before sharing

  • Keep the original unprotected file, test the protected output, and share the password through a separate trusted channel.
  • Do not use this as a legal redaction, certified security, password recovery, or compliance workflow.
  • For visible covers, signatures, or page cleanup before protecting, use PDF Workspace.
